Appendix A: Production of XML Schema for ORM XMI

 

Given a MOF-based ORM metamodel, it is possible to produce the formal XML definition for ORM XMI output. For this purpose an XML Schema was produced based on the foregoing MOF metamodel. A portion of the generated ORM XML Schema is shown below, representing a few of the elements and complex types of the ORM namespace. In addition to an ORM namespace definition, an associated XML Schema for the UML namespace is required, along with an XML Schema for the standard XMI base elements. Given the full XML Schema set, XMI generated from ORM information models can be validated for conformance to the ORM MOF metamodel.

 

<?xml version="1.0" encoding="UTF-8"?>

<xs:schema targetNamespace="ORM" xmlns:xs="http://www.w3.org/2001/XMLSchema" xmlns:UML="org.omg/UML1.4" xmlns="ORM" elementFormDefault="unqualified"  attributeFormDefault="unqualified">

  <xs:import namespace="org.omg/UML1.4" schemaLocation=".\UML14.xsd"/>

  <xs:element name="Type" type="TypeType"/>

  <xs:element name="ObjectType" type="ObjectTypeType"/>

  <xs:element name="EntityType" type="EntityTypeType"/>

  <xs:element name="FactType" type="FactTypeType"/>

  <xs:element name="Role" type="RoleType"/>

  <xs:complexType name="TypeType" abstract="true">

    <xs:complexContent>

      <xs:extension base="UML:ClassifierType">

        <xs:attribute name="isIndependent">

          <xs:simpleType>

            <xs:restriction base="xs:NMTOKEN">

              <xs:enumeration value="true"/>

              <xs:enumeration value="false"/>

            </xs:restriction>

          </xs:simpleType>

        </xs:attribute>

        <xs:attribute name="isExternal">

          <xs:simpleType>

            <xs:restriction base="xs:NMTOKEN">

              <xs:enumeration value="true"/>

              <xs:enumeration value="false"/>

            </xs:restriction>

          </xs:simpleType>

        </xs:attribute>

      </xs:extension>

    </xs:complexContent>

  </xs:complexType>

  <xs:complexType name="ObjectTypeType">

    <xs:complexContent>

      <xs:extension base="TypeType"/>

    </xs:complexContent>

  </xs:complexType>

  <xs:complexType name="EntityTypeType">

    <xs:complexContent>

      <xs:extension base="ObjectTypeType"/>

    </xs:complexContent>

  </xs:complexType>

  <xs:complexType name="FactTypeType">

    <xs:complexContent>

      <xs:extension base="TypeType">

        <xs:attribute name="sentence" type="xs:string"/>

      </xs:extension>

    </xs:complexContent>

  </xs:complexType>

  <xs:complexType name="NestedEntityTypeType">

    <xs:complexContent>

      <xs:extension base="FactTypeType"/>

    </xs:complexContent>

  </xs:complexType>

  <xs:complexType name="RoleType">

    <xs:complexContent>

      <xs:extension base="UML:StructuralFeatureType">

        <xs:attribute name="reading" type="xs:string"/>

      </xs:extension>

    </xs:complexContent>

  </xs:complexType>

</xs:schema>
